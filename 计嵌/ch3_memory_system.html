1. 存储器概述

层次化存储体系

离 CPU 越近 → 速度越快、容量越小、成本越高：

寄存器 → L1 Cache → L2 Cache → L3 Cache → 主存(DRAM) → 外存(HDD/SSD)
层次实现技术容量级别速度
寄存器触发器kB 级最快（CPU 内）
Cache（L1/L2/L3）SRAM32KB～数十MB纳秒级
主存（内存）DRAMGB 级纳秒级（较慢）
外存HDD/SSD/FlashTB 级毫秒级

性能指标

  • 存取时间 TA：从发出读命令到数据进入寄存器的时间
  • 存取周期 TM：连续两次独立操作的最小间隔，TM ≥ TA
  • 带宽 BM = W / TM（W 为总线位宽，单位 b/s）
⚠ 容量公式
若芯片地址线 n 根、数据线 m 根：
容量 = 2ⁿ × m 位 = 2ⁿ × m/8 字节
例：地址线13根、数据线8根 → 2¹³ × 8 bit = 8K × 8位 = 8KB

🔑 物理机制速记
  • 热电子注入（NOR Flash 写入、EPROM 编程）：漏源高压产生高能热电子，穿越 SiO₂ 注入浮栅 → 浮栅带负电 → 管子阈值升高 → 截止 → 存储"0"（或"1"视约定）
  • 隧道效应（NAND Flash 擦/写、EEPROM 擦/写）：超薄氧化层（<20nm）中，强电场让电子"量子隧穿"，可双向通过 → 实现浮栅充放电
  • 均属"非易失性"：断电后浮栅被 SiO₂ 包围，无放电通路，电荷长期保存
  • 它们具有读写功能仍称 ROM，因为正常工作状态下只读，写入需特殊高压或条件，且非易失性与 ROM 本义一致

NOR Flash vs NAND Flash

NOR Flash
  • 读：随机字节/字寻址，支持 XIP（芯片内执行）
  • 写：单字节/字编程（热电子注入）
  • 擦：以块为单位
  • 速度慢（擦写），适合存代码
  • 嵌入式系统启动 ROM
NAND Flash
  • 读：以页为单位（串行）
  • 写：以页为单位（隧道效应）
  • 擦：以块为单位，速度快（2ms vs 750ms）
  • 密度高、位成本低，适合大容量存储
  • U盘、SSD、eMMC、SD卡存储介质

3. RAM：SRAM 与 DRAM

特性SRAM（静态）DRAM（动态）
存储原理六管双稳态触发器（锁存器）单管 + 电容（靠电荷量存储）
刷新无需刷新需定时刷新（约 2ms/次，逐行）

典型应用Cache，少量嵌入式系统计算机主存（内存条）

4.1 位扩展（Bit Extension）

典型例题：用 8 片 1M×1位 芯片扩展为 1M×8位 的 RAM

图 A：位扩展示意（参考 图3.24）— 8片1M×1位 → 1M×8位
✅ 位扩展 — 规则总结
  1. 各芯片只有 1 根（或少量）数据线，分别连接总线的不同位（D₀、D₁…D₇）
  2. 所有芯片地址线 A₀～Aₙ 完全并联，接相同的 CPU 地址线
  3. 所有芯片 CS# 并联，R/W# 并联，同时被选中
  4. 结果：每个地址对应的 8 个芯片同时工作，合并输出 8 位数据

4.2 字扩展（Word Extension）

典型例题：用 4 片 256K×8位 芯片扩展为 1M×8位 的 RAM

分析：目标1M = 4 × 256K → 字数×4，位宽不变（8位）→ 纯字扩展

图 B：字扩展示意（参考 图3.25）— 4片256K×8位 → 1M×8位
✅ 字扩展 — 规则总结
  1. 各芯片数据线 D₇~D₀ 并联到数据总线（同一时刻只有一片 CS 有效，其余三态高阻）
  2. 低位地址线（片内地址）并联接所有芯片
  3. 高位地址线接译码器，译码器输出的各信号分别接不同芯片的 CS#
  4. R/W# 并联
  5. n 位高位地址线 → 2ⁿ 个芯片选择信号（即可扩展 2ⁿ 倍字数）

4.3 复合扩展（Composite Extension）

当芯片位宽和字数均不足时，先做位扩展组成一个"基本单元"，再对多个基本单元做字扩展。

📐 题3.27 示例：4片 4K×8位 → 8K×16位
1
分析需求：目标 8K×16位。芯片 4K×8位。
字数需扩展：8K / 4K = 2倍（需字扩展，用2组）
位宽需扩展：16位 / 8位 = 2倍（每组需2片位扩展）
共需：2组 × 2片/组 = 4片 ✓
2
位扩展：每组2片 4K×8位 → 4K×16位
• 片1：D₁₅~D₈（高8位）
• 片2：D₇~D₀（低8位）
• 两片地址线并联（A₀~A₁₁），同一 CS 信号控制
3
字扩展：2组 4K×16位 → 8K×16位
• A₁₂ 通过译码电路控制各组 CS#
• A₁₂=0 → 左组（#0, #1）选中，地址区间 0x0000~0x0FFF
• A₁₂=1 → 右组（#2, #3）选中，地址区间 0x1000~0x1FFF
• 外部总 CS 信号与 A₁₂ 送入译码器，译码输出控制各组片选

5. Cache 地址映射与转换 ⭐⭐⭐ 核心考点

前置知识：Cache 行（Cache Line）结构
字段用途备注
数据字段存储从主存复制的数据块典型大小 32B 或 64B
Tag（标记）记录该块在主存中的位置信息用相联存储器实现（全相联/组相联）
有效位（Valid）指示此行数据是否有效1 bit
脏位（Dirty）标记数据被 CPU 修改但未回写主存写回法（Write Back）才有
替换控制位供替换算法使用（如 LRU 计数器）位数由算法和路数决定

5.1 全相联映射（Full Associative）

核心规则

主存任意块可映射到 Cache 任意块。需要块表（相联存储器）记录全部映射关系。

地址格式

主存地址 = [ M位：块号 | W位：块内地址 ]
Tag = 全部 M 位（主存块号）
地址转换过程
①
CPU 给出主存地址，提取 M位块号
②
在相联存储器（块表）中查找是否存在此块号
③
若命中（Hit）：取出对应 Cache 块号 C，拼接块内地址 W → Cache 地址
④
若未命中（Miss）：从主存读取，同时将整块调入 Cache 某个空闲位置

优点：命中率高，Cache 利用率高；缺点：每次需与全部 Cache 块比较，电路复杂，仅适用小容量 Cache

5.2 直接相联映射（Direct Mapping）

核心规则

将主存按 Cache 大小分页，页内相同块号的主存块只能映射到 Cache 中相同块号的位置。

映射关系：Cache块号 i = 主存块号 j mod 2C

地址格式（关键！）

主存地址 = [ T位：页号 | C位：块号（=Cache块号） | W位：块内地址 ]
Tag = 页号 T 位（存放在对应 Cache 块的标记字段中）

其中： C = log₂(Cache总块数)， W = log₂(块大小/字节)， T = 总地址位数 - C - W

地址转换过程
①
从主存地址提取 C位（块号），直接定位到 Cache 第 C 块
②
读取该 Cache 块的 Tag，与主存地址的 T（页号） 比较
③
相符 → 命中，用块内地址 W 访问 Cache；不符 → 未命中，去主存取数并替换该 Cache 块

优点：硬件简单，仅需 1 次比较；缺点：不同页内同块号的块竞争同一 Cache 位置，冲突多，命中率低

5.3 组相联映射（Set-Associative）最常用

核心规则（折中方案）

Cache 分成 2u 组，每组 2v 块（称为 2v 路组相联）。
主存块 → Cache 的组采用直接映射（组号固定）；组内各块采用全相联（任意位置）。

地址格式（关键！）

主存地址 = [ s位：页号 | u位：页内块号（=Cache组号） | W位：块内地址 ]
Tag = 页号 s 位

其中： u = log₂(Cache组数)， v = log₂(每组块数) = log₂(路数)， W = log₂(块大小)， s = 总地址位 - u - W

地址转换过程
①
从主存地址提取 u位组号，定位到对应的 Cache 组
②
在该组的所有 2v 个块中，用相联存储器并行比较各块 Tag 与主存地址的 s 位页号
③
匹配成功 → 命中，取出对应块的 v 位块内编号，与 u、W 拼成 Cache 地址
④
无匹配 → 未命中，从主存调入，按替换策略（LRU）选择组内一块替换

5.4 三种映射方式终极对比 必考

对比维度 全相联 直接相联 组相联（k路）
主存地址格式 [M | W] [T | C | W] [s | u | W]
Tag 含义 全部块号（M位） 页号（T位） 页号（s位）
Tag 位数 最多（M位） 适中（T位） 最少（s位）
比较次数 与所有块比较（最多） 只比较 1 次（最少） 与组内 k 块比较
是否用相联存储器 是（大） 否（普通比较器） 是（小，仅组内）
命中率 高 低（冲突多） 较高
硬件复杂度 高 低 中
实际应用 极小容量TLB 少 现代CPU主流（8路、16路）
⚠ 特殊关系
组相联是通用形式，两者是其极端情况：
• 全相联 = 整个Cache作为1个组（u=0，v=C）
• 直接映射 = 每组只有1块（u=C，v=0）

5.5 替换策略 & 写入策略

替换策略（需全相联/组相联才有选择问题）

策略原则优点缺点
随机（Random）随机选一块替换 硬件简单被换出的块可能马上需要
LFU（最不经常使用）替换访问次数最少的块 考虑历史频率新调入的块易被替换（次数为0）
FIFO（先进先出）替换最先调入的块 实现简单常用块可能先进而被替换
LRU（近期最少使用）替换最长时间未被访问的块 命中率最高，最合理需计数器，稍复杂
💡 LRU 计数器位数
k 路组相联 → 需要区分 k 块的使用顺序 → LRU 位数 = ⌈log₂(k)⌉
例：8路组相联 → LRU 位 = 3位（可编码 0~7）

写入策略

写通法（Write Through）
  • CPU 写数据同时写 Cache 和主存
  • 简单，Cache 与主存始终一致
  • 每次写都要访问慢速主存，降低写速度
  • 无需脏位（Dirty Bit）
写回法（Write Back）
  • CPU 写数据只写 Cache，设置脏位=1
  • 仅当该块被替换时，才把修改的数据写回主存
  • 减少主存访问次数，写速度快
  • Cache 与主存可能不一致，有一致性隐患

6. 虚拟存储器（概要）

管理方式块的大小地址组成地址映射表优点缺点
段式不定长（按逻辑段） 段号 S + 段内偏移 D段表（段号、段长、起始地址、装入位、访问方式） 符合程序逻辑，易保护/共享 段间碎片，大段换入耗时
页式定长（4KB 等） 虚页号 P + 页内偏移 D页表（页号、实页号、装入位、修改位）+ TLB 快表 碎片小，大多数OS采用 不符合程序模块，保护共享不便
段页式段内分页（定长） 段号 S + 虚页号 P + 页内偏移 D段表 + 页表（两级查表） 兼顾两者优点 地址变换需三次内存访问
💡 TLB（快表）
页式管理中，每次访存需先查页表（位于主存），会使访存次数加倍。TLB 是对最活跃页表项的专用 Cache（相联存储器实现），硬件高速检索，避免频繁查主存中的慢表。
Cache vs 虚拟存储器 — 异同
维度Cache虚拟存储器
层次主存 ↔ Cache（SRAM）外存（磁盘）↔ 主存（DRAM）
块大小定长（32/64B）定长（页式4KB）或不定长（段式）
未命中代价低（ns级）极高（ms级，性能损失100~1000倍）
管理方式纯硬件，对所有程序员透明OS软件+硬件共同完成
共同点均利用局部性原理，均需地址转换，均可提升系统性能

第（2）部分 — 数据 Cache 缺失次数

1
确定数组 s 的大小与内存布局
数组 s：1024 个 int，每个 4B → 总大小 = 1024 × 4 = 4096B = 4KB
起始地址：0080 00C0H
2
检查起始地址的块内偏移
0080 00C0H 的低6位 = C0H & 3FH = 00H（全0！）
0x...C0 = ...1100 0000，低6位 = 00 0000 = 0
→ 数组 s 起始地址与主存块边界对齐（块内偏移=0，s[0]恰好在某个主存块的开始处）
3
计算数组 s 占多少个主存块
主存块数 = 4096B / 64B = 64 块
4
分析 Cache 缺失规律
循环对每个 s[k] 先读后写（s[k] = 2*s[k]）：
• 由于地址对齐，每个主存块恰好对应 64/4 = 16 个 int 元素
• 第一次访问某块的第1个元素（s[k]） → Cache 缺失（调入整块）
• 该块内后续15个元素的访问 → Cache 命中（块已调入）
• 32KB Cache 远大于4KB数组 → 整个数组可一次性全部放入 Cache，无替换
• 因此：每块恰好缺失1次（首次访问），共 64 块
5
统计缺失次数
Cache 缺失次数 = 64 次（每块首次访问时缺失1次）
✅ 第（2）小问答案

访问数组 s 的数据 Cache 缺失次数 = 64 次

原因：数组对齐块边界，共占64个主存块，每块首次访问缺失1次，Cache 容量足够容纳全部数组，无额外替换缺失。

💡 启示
对于追求高性能的程序，若数据能全部载入 Cache（数组大小 ≪ Cache 容量），代码运行效率极高。如本题中 4KB 数组远小于 32KB Cache，除首次各块的强制缺失（64次）外，后续循环中每次访问均命中。
